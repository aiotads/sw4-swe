# SKILL：SWE@Google 五階段研究閱讀流程

把一章原文（`chapters/raw/chNN.txt`）轉成兩份成品：研究筆記 `chapters/chNN.md`
與視覺化 `site/chNN.html`，全程遵循固定的研究流程與品質守則。**閱讀 CLAUDE.md
取得全書主軸、檔案結構與排版細節，本檔只規範「方法」。**

## 進度記憶與續做（每次開工先做）
跨 session 的進度記憶放在專案根的 **`PROGRESS.md`**（單一事實來源）。
- **開工前：** 先讀 `PROGRESS.md` 的「▶ 下一步（NEXT UP）」，得知該做哪一章與其圖況。
- **完成一章後（四個更新，缺一不可）：**
  1. `PROGRESS.md`：該章四欄（MD/EN/HTML/IDX）改 ✅、更新「▶ 下一步」指向下一章、
     更新總表完成數、在「變更日誌」加一行（日期用對話中的當前日期）。
  2. `CLAUDE.md` 的「進度」清單。
  3. `site/index.html`：該章卡片 `todo`→完成、加主軸標籤、更新進度條寬度與 `n / 25` 文字
     （中文與英文兩個版本都要改）。
  4. 回報使用者並等確認再進下一章（除非使用者要求連續產出）。
- 一律一次只做一章，確認後再續，確保隨時可安全中斷、下次無縫接續。

## 受眾與語氣（硬性）
- 對象：L4~L6 軟體工程師（初階到資深 / staff）。預設讀者懂程式，但不一定有大規模
  組織經驗。寫給「能做技術決策、需要理解 trade-off」的人。
- 深度：碩士等級以上的敘事觀點。**避免兩個極端**——不要淪為名詞解釋（太淺），
  也不要堆砌學術黑話（太艱深）。每個概念都要落到「工程師在現場會怎麼用」。
- 語言：繁體中文輸出。技術名詞、專有制度名（Hyrum's Law、Beyoncé Rule、SemVer、
  monorepo…）保留英文，首次出現附簡短中譯或註解。
- 翻譯原則：**保留語意，兼容母語文化與諺語轉意**。英文諺語/比喻（boiled frog、
  bus factor、shifting left）要譯出「神」而非逐字直譯，必要時加一句文化註解。
  原文核心主張不得在翻譯中失真或被稀釋。

## 五階段流程（每章都跑完整輪）

### 1. 廣讀 Skim — 建章節地圖
通讀 `chapters/raw/chNN.txt` 與 CLAUDE.md 中該章的小節清單，先掌握骨架：
這章在回答什麼問題？分幾個論證段落？最後的 TL;DRs 是什麼？
產出：心智圖骨架（章 → 小節 → 一句話主張）。

### 2. 精讀 Deep — 擷取論點與證據
逐節細讀，抓出：核心主張、支撐範例（Google 真實案例、數字、年份）、定義、
反例與警告。**所有數字與案例必須回到原文查核**（如「factor of 100,000」、
「2006 compiler upgrade」、「60–70% build locally」）。
產出：細節重點（依小節組織）＋ 待引用的原文金句清單（含原文頁碼語境）。

### 3. 思考 Reflect — 對 L4~L6 的決策意涵
把論點轉成「對你的工程決策意味著什麼」。扣回全書三大主軸：
**Time（時間）、Scale（規模）、Trade-offs（取捨）**。
產出：「為什麼要讀這一篇」（用讀者痛點切入，不是書評）＋ 取捨/反模式分析。

### 4. 提問 Question — 蘇格拉底式自我詰問
針對本章生成 3~5 題能驅動實作反思的問題（不是是非題），例如
「你的系統若套用 Hyrum's Law，哪些『未承諾但被依賴』的行為最危險？」
產出：延伸提問清單。

### 5. 重複閱讀 Review — 跨章連結與校驗
回扣其他章節（用 `[[chNN]]` 心智指向）、檢查中英一致性、確認沒有過度簡化或誤譯。
產出：跨章連結段落 ＋ 一次全文校對。

## 雙語（中／英可切換，硬性）
每章同時產出中文與英文兩個完整版本，HTML 內用**純 CSS 切換、零 JavaScript**：
- **Markdown：** `chapters/chNN.md`（繁中研讀筆記）＋ `chapters/chNN.en.md`（英文研讀筆記）。
  兩份都跑完整七段結構，內容對位但**不是逐句互譯**：
  - 中文版：碩士級繁中敘事；金句區 = 英文原文 + 繁中翻譯 + 中文語境註。
  - 英文版：清楚、帶解釋性的英文，用「**English-to-English**」方式幫助理解——把原書
    密集的論證用淺白英文重述、定義術語、補背景；金句區 = 英文原文 + 英文淺白釋義
    （paraphrase）+ 英文語境註。對稱於中文版「引用英文原文」的作法。
  - 英文版受眾同為 L4~L6，但服務「想用英文吸收、或想對照原文語感」的讀者，不得只是
    把中文機械翻譯回去，要讀起來像英文母語的學習導讀。
- **HTML 切換機制（CSS-only）：** `<body>` 開頭放兩個 `name="lang"` 的隱藏 radio
  （`#lang-zh` 預設 checked、`#lang-en`），右上角固定 `.lang-switch` 用 `<label>` 切換。
  所有語言相依文字包進 `<span class="zh">…</span>`／`<span class="en">…</span>` 或對應的
  區塊 `<div class="zh">／<div class="en">`。CSS：預設 `.en{display:none}`；
  `#lang-en:checked ~ * .en{display:…}` 顯示英文、`~ * .zh{display:none}` 隱藏中文。
  圖（SVG）的文字標籤同樣用 `.zh/.en` 兩套 `<text>` 切換；`<figcaption>` 雙語。
  切換鈕的 active 樣式由 `#lang-xx:checked ~ * label[for=lang-xx]` 控制。
- `index.html` 同樣加中/英切換（標題、說明、卡片標題雙語；英文卡片用英文章名與英文一句話）。

## Markdown 成品結構（chapters/chNN.md，固定七段）
依序輸出，標題用 `##`：

1. **摘要** — 150~250 字，這章在說什麼、結論是什麼。
2. **為什麼要讀這一篇** — 從 L4~L6 的真實痛點切入，3~4 點，說明讀完能改變什麼決策。
3. **細節重點** — 依原文小節組織，每節：核心主張 + 關鍵證據/案例 + 工程意涵。
   用子標題 `###` 對應原文小節名（中英並列）。
4. **原文引用** — 精選 3~6 句金句。每句格式（英文原文在上、繁中在下）：
   ```
   > **EN:** "<original sentence>"
   > **ZH:** 「<繁中翻譯，保留語意與諺語轉意>」
   > *語境/註解：<一句說明出處小節或文化背景>*
   ```
5. **關鍵字補充** — 5~10 個術語/制度名，每個一行：`**Term（中譯）** — 定義 + 為何重要`。
6. **延伸提問** — 階段 4 的 3~5 題。
7. **跨章連結** — 與哪些章互為前後文，用 `[[chNN]] 章名 — 關係一句話`。

開頭加 YAML frontmatter：`chapter`、`title_en`、`title_zh`、`part`、`author`、
`pdf_pages`、`themes`（time/scale/tradeoffs 取用到的）。

## 圖的處理（硬性政策）
原書每章可能有 `Figure X-Y`。先用 `_figs.py` 盤點，再分流：
- **概念圖／流程圖／圖表**（壽命光譜、測試金字塔、建置依賴圖、CI 流程…）：用
  **inline SVG** 在深色主題重繪，忠實還原該圖傳達的「資訊與關係」，配色沿用三主軸與
  全書 CSS 變數。每張圖包進 `<figure>`，下方 `<figcaption>` 標註對應 `Figure X-Y`
  與一句繁中圖說。SVG 用 `viewBox` 自適應、文字用 `currentColor`/主題變數。
- **產品 UI 截圖**（如 ch17 Code Search、ch19 Critique）：截圖無法重繪，用 PyMuPDF
  抽原圖轉 base64 內嵌，放進 `<details><summary>原圖 Figure X-Y</summary>…</details>`
  可收合區塊（`<details>` 為原生、不需 JS）；主視覺另用乾淨 SVG 流程示意還原其運作邏輯。
- **無正式 Figure 的章節**：可自行設計 1~2 張「概念整理 SVG」（如分類矩陣、決策樹）強化
  視覺化，但不得杜撰原書沒有的數據。
- 抽原圖指令：`page.get_images(full=True)` + `fitz.Pixmap`，輸出到 `site/assets/imgNN/`
  或直接 base64；務必確認抽出的是正確 Figure（比對前後文字）。

## HTML 成品（site/chNN.html）
- **精簡靜態**：單一自包含檔案，內嵌 `<style>`，**不用 JavaScript、不依賴外部 CDN**。
  唯一例外：UI 截圖以 base64 內嵌（仍屬自包含）。
- 套用 CLAUDE.md 規範的共用主題（深色學術風、同一套 CSS 變數與版面）。
- 區塊順序對齊 Markdown 七段，但視覺化呈現：
  頁首（章號/中英標題/Part/作者/頁碼徽章）→ 摘要卡 → 為什麼要讀（重點卡列）→
  細節重點（小節卡片，主張用強調色）→ 原文金句（中英並陳 blockquote）→
  關鍵字詞彙表（definition grid）→ 延伸提問（編號列表卡）→
  跨章連結（chips）→ 頁尾導航（上一章 / 索引 / 下一章）。
- 用純 CSS 表現「視覺化」：重點卡、主軸標籤色塊、簡單關係用 CSS grid/flex 排版，
  避免複雜 SVG。Time/Scale/Trade-offs 三主軸固定配色，全書一致。
